# Lab2
ОТЧЕТ ПО ЛАБОРАТОРНОЙ РАБОТЕ
Реализация классов в C# для различных сущностей
1. Цель работы
Разработать систему классов для представления различных сущностей (Человек, Имя, Город, Дробь) с соблюдением принципов объектно-ориентированного программирования и организовать демонстрацию их работы.

2. Выполненные задачи
2.1. Класс Person (Человек)
Цель: Создать сущность для представления человека с именем и ростом.

Реализация:

csharp
public class Person
{
    public string Name { get; set; }
    public int Height { get; set; }
    
    public Person(string name, int height) { ... }
    public override string ToString() { ... }
}

2.2. Класс FullName (Имя)
Цель: Реализовать сущность для хранения полного имени с поддержкой необязательных параметров.

Реализация:

csharp
public class FullName
{
    public string LastName { get; set; }
    public string FirstName { get; set; }
    public string MiddleName { get; set; }
    
    public FullName(string lastName, string firstName = null, string middleName = null) { ... }
    public override string ToString() { ... }
}
Особенности: Метод ToString() формирует строку только из заданных параметров, игнорируя null значения.

Результат: Созданы имена:

Маяковский Владимир

2.4-2.5. Класс City (Города)
Цель: Создать сущность для представления города как вершины графа с возможностью хранения связей с другими городами.

Реализация:

csharp
public class City
{
    public string Name { get; set; }
    public Dictionary<string, int> ConnectedCities { get; set; }
    
    // Два конструктора
    public City(string name) { ... }
    public City(string name, Dictionary<string, int> connectedCities) { ... }
    
    public void AddConnection(string cityName, int cost) { ... }
    public override string ToString() { ... }
}
Особенности:

Два конструктора (с названием и с названием+связями)

Использование Dictionary для хранения связей "город → стоимость"

Метод ToString() выводит связи в удобном формате

Результат: Создана схема из 4 городов с различными стоимостями путей между ними.

4. Основные принципы реализации
4.1. Инкапсуляция
Все поля классов инкапсулированы через свойства

Для дробей числитель и знаменатель доступны только для чтения извне

Внутренние методы (Simplify, GCD) объявлены как private

4.2. Наследование и полиморфизм
Все классы переопределяют метод ToString()

Демонстрация полиморфизма через перегрузку методов (несколько конструкторов, разные версии арифметических операций)

4.3. Обработка исключений
Проверка знаменателя на ноль при создании дроби

Обработка деления на ноль в операции Div

Проверка входных параметров
